## 数据结构阶段

### **第1讲：Redis入门与Go生态对接**
**内容边界：**
- Redis核心概念、Docker部署、CLI基础操作
- 单线程+IO多路复用架构
- go-redis客户端初始化、配置与基础CRUD
- **实战**：搭建Go+Redis环境，实现`Ping`检测

**总结**：理解Redis定位，掌握Go项目集成方式

---

### **第2讲：String类型与缓存模式**
**内容边界：**
- String命令（`GET/SET/MSET/INCR/SETEX`）
- 应用场景：缓存、计数器、分布式锁
- **实战**：高性能**文章阅读计数器**（原子递增+过期策略）
- 缓存设计模式：Cache-Aside、防穿透初步

**总结**：String作为最常用类型的选型原则

---

### **第3讲：Hash类型与对象建模**
**内容边界：**
- Hash命令（`HGET/HSET/HGETALL/HINCRBY`）
- 应用场景：结构化数据存储、用户信息
- **实战**：**用户资料管理系统**（对比String存储的优劣）
- 数据建模原则：何时用Hash而非String

**总结**：掌握对象粒度控制与内存优化技巧

---

### **第4讲：List类型与队列模式**
**内容边界：**
- List命令（`LPUSH/RPOP/BLPOP/LRANGE/LLEN`）
- 应用场景：消息队列、最新动态、任务调度
- **实战**：**异步任务队列**（生产者-消费者+Go协程）
- 队列选型对比：Redis List vs 专业MQ

**总结**：理解List作为轻量级队列的适用边界

---

### **第5讲：Set类型与关系运算**
**内容边界：**
- Set命令（`SADD/SMEMBERS/SINTER/SUNION/SPOP`）
- 应用场景：标签系统、好友关系、抽奖去重
- **实战**：**文章标签系统**（标签添加、交集查询、随机推荐）
- 集合运算性能考量

**总结**：掌握Set在关系计算中的独特价值

---

### **第6讲：Sorted Set与排行榜设计**
**内容边界：**
- Sorted Set命令（`ZADD/ZRANGE/ZREVRANK/ZINCRBY`）
- 应用场景：排行榜、延迟队列
- **实战**：**游戏积分排行榜**（实时排名、Top10、段位查询）
- Score设计技巧：时间戳编码、多维度权重


**总结**：理解Sorted Set是唯一可排序集合的优势

---

### **第7讲：高级类型与HyperLogLog应用**
**内容边界：**
- Bitmap、HyperLogLog
- **实战**：**用户签到与DAU统计系统**（Bitmap签到，HyperLogLog统计）
- 内存对比：HyperLogLog vs Set


**总结**：掌握特殊场景下的降本增效方案

## 通信和内核机制阶段
### **第8讲：Reids Stream与消息队列**
**内容边界**
- Redis Stream 使用
- 生产者消费者，消费者组概念
- 可靠的Go消息队列
- Redis Stream 与专业 MQ 的边界

**实战**
- 编写可靠 Go 消息队列

**总结**
- Redis Stream实现简单可靠的消息队列

### **第9讲：Redis发布订阅与实时通信模型**

**内容边界：**

* Pub/Sub模型原理
* `PUBLISH / SUBSCRIBE / PSUBSCRIBE`
* Channel设计规范
* Go客户端订阅模型（阻塞、协程、重连）
* 发布订阅 vs Stream vs MQ 对比
* 实时系统设计模式

**实战：**
* 实现Go订阅实时通知

**总结**
- 理解Redis Pub/Sub是实时通信工具，而不是可靠消息队列



### **第10讲：Redis核心机制（持久化与事务）**

**内容边界：**

* 持久化机制

  * RDB原理、触发方式、优缺点
  * AOF日志模型、重写机制
  * RDB vs AOF选型策略
* Redis事务机制

  * `MULTI / EXEC / DISCARD`
  * `WATCH`乐观锁机制
  * 事务执行流程与原子性边界
* 事务与Lua的关系

**实战：**

* 编写Go脚本测试RDB/AOF恢复
* 使用WATCH实现库存扣减示例

**重点陷阱：**

* Redis事务 ≠ ACID事务
* 无回滚机制
* 命令入队错误 vs 执行错误

**总结：**
理解Redis数据安全与一致性边界，避免把Redis当关系数据库使用。
